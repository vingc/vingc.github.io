optimization-of-ssh-agent-and-ssh-keys-for-git-bash

Several days ago, i firstly installed git in my new laptop, and encountered a problem about management of ssh-agent and ssh keys.

After started a new “git bash” terminal, there was something wrong when pushed my new commit, see below picture:

when used command “ssh-add -l” to check the private keys added, it returned an error.

After google, realized that “git bash” needs we start the “ssh-agent” process before “ssh-add” and “git” command to use ssh channel.

Lay why “git bash” doesn’t launch “ssh-agent” when the terminal launched.
